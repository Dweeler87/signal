How it works.

Forelight runs its own Certificate Transparency log follower — no scraping, no polling, no lag.

1

CT logs emit every certificate in real time

Every publicly trusted CA must log certificates under RFC 6962. Forelight reads those logs continuously — every cert issued anywhere on the internet appears within minutes.

2

We parse, enrich, and type each event

Raw cert metadata → domain extraction → DNS/ASN/hosting enrichment → SaaS vendor detection → firmographic data (company name, industry, size). Classified into seven signal types and scored 1–100 by buying-signal strength.

3

Deliver via API or webhook

Pull on demand with filters (industry, SaaS vendor, hosting provider, since timestamp) or push to your endpoint via webhook. Clay-ready flat JSON — no nested objects.

Why not just use intent data?

Existing intent data is lagged, probabilistic, and largely the same Bombora co-op resold many times. Forelight is different by design.
